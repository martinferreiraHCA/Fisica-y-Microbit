ⓘ Modo Solo Variables
La micro:bit envía solo valores de variables (sin timestamp). El tiempo se registra con el reloj del navegador.
Formato: var1,var2,...,varN
ⓘ Submuestreo por video
Al activar este modo, cuando detengas la captura se conservarán solo los datos que coinciden con cada fotograma del video (1 dato por frame, ~30 por segundo).

¿Por qué? La micro:bit puede enviar datos a 50-100 Hz o más, pero el video graba a 30 FPS. Los datos intermedios no tienen un fotograma asociado, así que al reproducir el video cuadro a cuadro, esos datos "extras" no se pueden visualizar.

Ventaja: Cada punto de tu tabla y gráfica corresponde exactamente a un cuadro del video. Ideal para análisis de video.
Compromiso: Se descartan los datos recibidos entre fotogramas. Si necesitas máxima resolución temporal, deja este modo desactivado.

Código base micro:bit

Tiempo + Variables (por defecto):

basic.forever(function () {
    let sensor = pins.analogReadPin(AnalogPin.P0)
    serial.writeLine("" + input.runningTime() + "," + sensor)
    basic.pause(100)
})

Solo Variables (sin timestamp):

basic.forever(function () {
    let x = pins.analogReadPin(AnalogPin.P0)
    let y = pins.analogReadPin(AnalogPin.P1)
    serial.writeLine("" + x + "," + y)
    basic.pause(100)
})

El formato de cada línea debe ser:
tiempo,var1,var2,... o var1,var2,...
